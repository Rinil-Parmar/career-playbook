# Java OOP Interview Question Bank

Use this as a progressive practice set for co-op interviews.

## Theory Questions

1. Explain encapsulation with a real Java class example.
2. Difference between abstraction and encapsulation.
3. Difference between interface and abstract class.
4. What is polymorphism? Give compile-time and runtime examples.
5. Composition vs inheritance: when should each be used?
6. Why do `equals()` and `hashCode()` need to be consistent?
7. What does `final` mean for class, method, and variable?
8. Why is immutability useful in concurrent systems?
9. What is method overriding vs method overloading?
10. Why are access modifiers important in API design?

## Coding Questions (Common Interview Style)

1. Design a `BankAccount` class with validation, deposit, withdraw, and transfer.
2. Design an `Employee` hierarchy (`FullTimeEmployee`, `Contractor`) and compute pay polymorphically.
3. Build a `Shape` interface with `Circle`, `Rectangle`, `Triangle` implementations and area calculation.
4. Design a simple `Library` system with `Book`, `Member`, and `LibraryService` using composition.
5. Implement immutable `Money` class (`amount`, `currency`) with safe operations.
6. Create a notification system using interface-based design (`EmailNotification`, `SmsNotification`).
7. Refactor a class using inheritance into a composition-based design.
8. Implement custom `equals()` and `hashCode()` for a domain model class.

## Practice Strategy

1. Solve one theory + one coding question daily.
2. For coding answers, explain class responsibilities before writing code.
3. After solving, identify one improvement for extensibility.
